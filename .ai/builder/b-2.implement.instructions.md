# Constructor: Instrucciones para la implementación de funcionalidades

## Rol

Actúa como _ingeniero de software_ para implementar las **tareas específicas** de un plan de implementación previamente generado.

## Proceso

1. **Configuración de la plantilla**

Lee los siguientes documentos:
- [Sintaxis de la plantilla](../../.ai/syntax.template.md) para comprender cómo usar las plantillas.


2. **Recopilación de información**


- Lee el Plan de Implementación en `docs/features/{{feature.slug}}/plan.md` para obtener las instrucciones de las tareas.
- Consulta los documentos de mejores prácticas: [BEST_PRACTICES.md](../BEST_PRACTICES.md) y [PATTERNS.md](../PATTERNS.md)
- Para cada tarea en el archivo de plan:
  - Implementa la tarea paso a paso siguiendo el orden numérico.

3. **Resultado**

- El **código fuente** para todas las tareas del plan:

**Criterios de Validación** (ver [QUALITY_GATES.md](../QUALITY_GATES.md#gate-5-code-implementation-completion)):
- Todas las tareas del plan están implementadas
- Código sigue los patrones arquitectónicos especificados en el plan
- Código adhiere a los estándares de codificación del proyecto
- Análisis estático pasa (linters, type checkers, etc.)
- Código compila/ejecuta sin errores
- Código está organizado siguiendo la estructura del proyecto
- Dependencias están correctamente gestionadas
- No hay valores hardcodeados o números mágicos (usar constantes/config)
- Si el plan incluye capa de Presentación, todos los componentes UI están implementados (templates, vistas, formularios, componentes reutilizables, etc.)
- La interfaz de usuario sigue los requisitos UI/UX definidos en la feature

**Recursos de Referencia**:
- Consulta [PATTERNS.md](../PATTERNS.md) para patrones arquitectónicos y de diseño
- Consulta [BEST_PRACTICES.md](../BEST_PRACTICES.md) para mejores prácticas de desarrollo
- Sigue las convenciones del lenguaje y framework utilizados en el proyecto

- El **código fuente** para todas las tareas del plan:
  - Para cada tarea:
    - Piensa en 2 o 3 opciones de implementación.
    - Evalúa la complejidad de la implementación.
    - Evalúa los cambios delta necesarios en el código existente.
    - Elige la opción más simple.
    - Escribe el código para esta tarea en el archivo(s) apropiado(s).
    - Ejecuta las herramientas de análisis de código estático del proyecto (linters, formatters, type checkers) y corrige cualquier error.
    - Verifica que el código compile/ejecute y funcione correctamente según las convenciones del lenguaje y framework utilizado.

4. **Seguimiento**

- Actualiza la incidencia de GitHub de la funcionalidad:
  - Cambia el estado a `builder-planned` cuando comiences
  - Cambia el estado a `builder-implemented` cuando completes todas las tareas
  - Añade comentarios con el progreso de las tareas implementadas
- Confirma los cambios utilizando el siguiente formato convencional de mensaje de confirmación:
  - `feat: Implment {{feature.slug}} closes #<issue-number>`
